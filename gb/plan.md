# ゲストブック要件まとめ

## ページ構造

- 1カラム
- 最上部にフォーム、下に投稿一覧（最新順）
- フォームと投稿一覧は細い区切り線で分離

---

## フォーム

### 入力項目

- 名前（任意、空欄時は `Anonymous`）
- 件名（任意、空欄時は `(no subject)`）
- 本文（必須）

### ボタン

- 投稿ボタン
- クリアボタン（即クリア・確認なし）

### フロー

- 投稿前にプレビュー確認画面
- 投稿後に時間帯で変化するありがとうメッセージ

---

## 返信ボタン

各投稿に設置。押すと：

- 件名欄に `re: [元の件名]` または `re: (no subject)` を上書き
- 本文先頭に `>>No.\n` を挿入（既存テキストがあれば先頭に追加）
- フォームへスクロール・本文欄にフォーカス
- `re:` の連鎖はそのまま許容

---

## 投稿の表示

### 表示例

```
#003
From    : yuinoid ★
Date    : 2026-05-03 03:14  [1746234840]
Subject : re: はじめまして

>>002 来てくれてありがとうございます。

48 chars / 93 bytes
```

### 表示項目

| 項目 | 備考 |
|------|------|
| No. | 通し番号 |
| From | 空欄時は `Anonymous`、管理者は `★` マーク |
| Date | 人間向け日時 + UNIXタイムスタンプ併記 |
| Subject | 空欄時は `(no subject)` |
| 本文 | |
| 文字数・バイト数 | 例: `48 chars / 93 bytes` |
| ID | 同IDの投稿を辿れるリンク |

---

## レス番号・ID

- `>>No.` 記法はアンカーリンクとして機能
- 存在しない未来のNo.は薄く表示
- IDもリンクとして機能（同ID投稿の一覧）

---

## セマンティック・技術的要素

### microformats2（h-entry）

各投稿に適用する基本構造：

```html
<article class="h-entry">
  <a class="u-uid" href="#003"></a>
  <span class="p-author">yuinoid</span>
  <time class="dt-published" datetime="2026-05-03T03:14:00+09:00">
    2026-05-03 03:14
  </time>
  <data class="p-name" value="re: はじめまして"></data>
  <div class="e-content">
    来てくれてありがとうございます。
  </div>
</article>
```

返信の場合は `u-in-reply-to` を追加：

```html
<a class="u-in-reply-to" href="#002"></a>
```

### HTMLセマンティクス

- `<time datetime="...">` にUNIXタイムスタンプを `<data value="...">` で併記
- 投稿を `<article>`、ヘッダー部を `<header>` で構造化
- IDリンクに `rel="tag"` を付与

### RSSまたはAtomフィード

- 投稿をフィードとして配信
- 各投稿がfeedのitemになる

## セキュリティ

| 対策 | 実装方針 |
|------|----------|
| XSS対策 | 本文のHTMLエスケープ |
| スパム対策 | honeypotフィールド |
| レート制限 | 同一IPからの連投防止 |
